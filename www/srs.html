Introduction

This is the senior design project for the Informatics program at UCI. Our goal is to make an open source project hosting solution based off the Google Code Collaborative Development Environment (CDE). This system will be installed on a project's own servers instead of running on Google's servers. In the short term we will be porting the system from Google proprietary technologies to open source solutions. In the future we will introduce some new ideas based off feedback and observations from the open source community.

We hope that this project is not only a single successful open source project, but also produces a system that benefits the open source community as a whole. After June 2008 we will continue this project and promote it as a fully-fledged open source project by getting the word out and allowing other people to contribute their own design, coding, and testing skills.

Design Guidelines and Environmental Constraints

Project development constraints

  • This project with its associated documents and code will be hosted as a project on code.google.com.
  • The entirety of the code for this project will be open source under the Apache 2.0 license.
  • The project is based on the existing Google Code project. See the design document for information about how this influences the design.

Environment

  • This system will run on a mid-level server running a Unix environment.
  • Subversion is assumed to have already been installed on this server, but we make no further assumptions about the nature of the subversion installation or configuration.

Language

  • The system will be written in Python 2.5.1

Non-Functional Requirements

Reliability

  • The system shall be reliable. System downtime could have a very detrimental effect on the projects hosted using this system and should be minimized. Under no circumstances will accidental data malformation or loss caused by the system be acceptable. However, data malformation or loss could still happen if the system's interface is bypassed, an issue brought up in the "robustness" requirement below.

Compatibility

  • All rendered pages shall render acceptably well on the major browsers being used on the Internet. This includes Firefox, Safari, Opera, Netscape, and Microsoft Internet Explorer 6 and 7. By acceptably well we acknowledge that pages will not render identically across all browsers, but what is rendered shall still be usable.

Robustness

  • The system shall be robust. Since users will be allowed to make changes to their project(s) by bypassing the web interface (such as by using a third party subversion client) there is the potential for the project to contain malformed or missing data. The system will gracefully handle data integrity issues.

Open Source

  • This project will act as a good example of open source methodologies and practices, as laid out in [1] and [2].
  • After completion of the Informatics 191 course in June 2008, development of the system will continue indefinitely as a fully fledged open source project (still under the Apache 2.0 license). Contributers other than the four individuals taking the course will be welcome and encouraged to make contributions to the project.
