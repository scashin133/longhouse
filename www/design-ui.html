Overview

What are the most important facts that a developer should know about the user interface of this system?

The templates for this system were originally obtained from the Google code project. We tried to maintain the original UI as much as possible to make it easier for people to use. This is going with the thought that the people that are using this system have used Google Code. However, as you will notice we have made some, what we will call, improvements to the existing UI. The most noticeable being the new form instructions.

What are the ranked goals for the user interface of this system?
  1. Task support and efficiency
  2. Consistency and familiarity

Technical Constraints / Operational Contextualization

What are your assumptions about the output devices?
We make very few assumptions about the user's screen or web browser, other than the assumption that they can view page somehow. We do assume that javascript is enabled.
What are your assumptions about the input devices that you will use?
We assume only that the user has a standard keyboard and mouse.
What windowing systems, UI libraries, or other UI technologies will you use?
HTML and CSS with GIF and PNG images.
Using Prototype and Scriptaculous for javascript
