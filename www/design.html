Introduction

How is this design document organized?

This document represents the two phases of designing this system:
  1. Recovery of the existing system. This is no easy task for two reasons. First, we were only given part of the code for the existing system, which does not run. This was necessary because the code contains proprietary Google technology which they did not wish to open source. A second reason is that it is written in Python, an interpreted scripting language. This means that the code does no compile-time checking, for instance ensuring that data types are compatible or that methods called on an object actually exist. The net result of these two challenges is that it is both difficult to determine what pieces of the system we do not have and the nature of those pieces.
  2. Design of the new system. After recovering the design of the code we were given we can determine how to write our system. Some parts of the code we were delivered can be reused and some must be modified. Additionally we must write new code to act as the "glue" between the existing code and the open source technologies we are using, for instance classes to interface with the Twisted web server.

What are the most important facts that a developer should know about this design?

This design is highly reflective of the design for Google Code. The reason for this is that we were given a portion of the codebase for Google Code to use as a starting point of our project. We have two reasons for wanting our design to remain as similar to Google Code's design as possible:
  • We would like to reuse as much code as possible rather than rewrite it so we can have this system built and running as fast as possible.
  • We would like changes to the system to be easily exchanged in both directions. That is, new features implemented in Google Code can easily be integrated into our system, and new features we add can be easily integrated into Google Code.
A key difference between this system and Google Code is we replaced Google Code's proprietary web server with an open source one, Twisted.
