Agreed Goals

After several communications with stakeholders the following is the agreed upon project goal:

Our goal is to make an open source project hosting solution based off the Google Code Collaborative Development Environment (CDE).

Environment

What is the system's physical environment?
This system makes no assumptions about the type of web server that it is being hosted on. The physical environment should be accommodated no matter what situation.
What is the system's technology environment (hardware and software)?
The hardware should be a mid-level server.
The system will be a unix operating system. The system will be running Python and will require a subversion repository.

Stakeholders / Actors

All
All stakeholders share the following key needs:
  1. Easy to use interface
Open Source Software User
A user of open source is going to want to know details about a open source project quickly and with minimal effort. Things like how active the project is, the purpose of the project, what stage of development it is in. They also will want to report about what is wrong with the software and be able to receive updates.

Key needs:

  1. Ability to search for a project that does what they need.
  2. Needs to be able to process information about a project quickly
  3. Send issues to a project
Open Source Software User > Project Member
A member of a project covers a lot of ground. It goes from people who contribute code to people who update documentation. Their experience in development will also vary from a beginner in open source development to more advanced.

Key needs:

  1. Contribute code and documents
  2. Updates about current development
  3. Receive and resolve issues
Project Member > Project Owner
Project owner is more or less a project member but with the responsibility of maintaining the project as a whole. They will be very experienced in open source development and will most likely be involved in multiple projects or at least will have been involved.

Key needs:

  1. Easy interface to do it all in
  2. Edit Project Details
  3. Add/Remove Project Members
Project Owner > System Administrator
These are the people who have installed our system. They will be in charge of maintaining all projects. They are expected to be experienced in Unix enough to be able to install our system and also maintain it.

Key needs:

  1. Easy installation
  2. Maintain multiple project installations
Professor Ziv and Sameer
Although they may never actually use the product it is still very important to them that the project succeed. They are the people who coordinated our project and talked to Google about starting up a project. They will help coach us into making better documents and help to keep up on track.

Key needs:

  1. Customer stays happy
  2. Have a working displayable result by the end of each quarter and a working product by the end of the year
Jason (Customer)
Works for Google and is helping us to make sure that the project stays on course and within vision. He has a great amount of domain knowledge when it comes to open source and will be sharing it with us to help keep the project moving forward.

Key needs:

  1. Turn into thriving open source project
