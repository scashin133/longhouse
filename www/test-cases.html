TC-101A: Successful Login

Purpose: To verify that users can successfully login to an existing account using their known name and password.
Prereq: User has created an account with the system with a known username and password.
Steps:
  1. Click "login" on any page of the system.
  2. Enter user name and password.
  3. Verify that the "login" link is now replaced with a status message indicating that the user is logged in, and a "logout" link is provided.

TC-101B: Rejection Due to Lack of User Approval

Purpose: Verify that the system rejects users with incorrect username/password combinations.
Prereq: User has created an account with a known username and password.
Steps:
  1. Click "login" on any page of the system.
  2. Enter the correct username and a password different than the known password.
  3. Verify that the system rejects the login attempt.

TC-103A: Successful Logout

Purpose: Determine whether the system correctly logs out a user when the logout option is selected.
Prereq: Log in successfully (see TC-101A).
Steps:
  1. Click "logout" on any page of the system.
  2. Verify that the system now does not show the user as logged in using his or her username, and instead provides a "login" link.

TC-200A: Successful Creation

Purpose: Determine whether new projects are createable.
Prereq: None
Steps:
  1. Log in (see TC-101A).
  2. On the main system page, choose to create a new project.
  3. Enter all the fields required to create a project.
  4. Submit the request to create a project.
  5. Verify that the system indicates when the project creation is complete, and takes the user to the new project's home page.

TC-200B: Already-Used Name

Purpose: Determine whether the system allows a new project to use the same name as another project.
Prereq: Create at least 1 prior project (see TC-200A).
Steps:
  1. Log in (see TC-101A).
  2. Choose to create a new project.
  3. Fill out all data for the new project. For the project name, choose the exact same name as the name of the project previously created.
  4. Submit the request to create the new project.
  5. Verify that the system disallows the request to create a new project and forces the user to choose a different name.

TC-200C: Missing Information

Purpose: To determine whether the system does or does not allow projects with incomplete information to be created.
Prereq: Have an account.
Steps:
  1. Log in (see TC-101A).
  2. Choose to create a new project.
  3. Enter all information for the new project, but leave the Project Name field blank.
  4. Verify that the system disallows the creation of the project and forces the user to give a project name.
  5. Repeat for other fields, leaving the Summary, Description and License fields blank in succession.

TC-200D: User Has Already Created 10 Projects

Purpose: Determine whether the system allows users to create projects beyond the stated limit.
Prereq: Create 10 prior projects (see TC-200A).
Steps:
  1. Log in (see TC-101A).
  2. Choose to create a project.
  3. Enter all information for the new project and submit the creation request.
  4. Verify that the system does now allow the project to be created because the user's project creation maximum has already been reached.

TC-200E: Disk Space is Full

Purpose: To determine whether projects can be created when the disk is full.
Prereq: Fill the entire hard disk that the server is running on.
Steps:
  1. Log in (see TC-101A).
  2. Choose to create a project.
  3. Enter all information for the project.
  4. Submit the project creation request.
  5. Verify that the system rejects the request to create the project because the disk is full.

TC-201C: No Results

Purpose: Verify that the search engine gives no results correctly, when there is no project matching a search query.
Prereq:
  1. Ensure that there is no page in the system with a given word (can be either a real or nonsense word).
  2. Ensure that there is one label or more that does not have a project associated.
Steps:
  1. On the main system page, search by the word that should not be found anywhere in the system.
  2. Verify that no search results appear.
  3. On the main system page, browse by the label that has no projects associated.
  4. Verify that no projects appear in the list of projects associated with that label.

TC-300A: Project Settings Change via Web UI

Purpose: Determine whether the project owner can change project settings using the Web UI.
Prereq: User is logged in (see TC-101A).

  1. Enter the project settings portion of the website by clicking on the "Administer" tab.
  2. Change the contents of the field, "Summary."
  3. Click "Save Changes."
  4. Click the "Project Home" tab.
  5. Verify that project summary on the project home page reflects the change made.
  6. Repeat steps 1-5 with the Project Description field, the license and the project labels.
  7. Attempt to add a new link to the Links list by clicking "Add row" and then supplying a text and URL.
  8. Repeat steps 3-5.
  9. Repeat steps 7-8 for the Blogs link list.
  10. Attempt to add a new discussion group by supplying a description and an email address.
  11. Repeat steps 3-5.
  12. Repeat steps 10-11 for the Activity Notifications.

TC-400A: File List Manipulation

Purpose: Determine whether manipulating the file list in the Downloads section works as intended.
Prereq: Have at least two or more files uploaded to the Downloads section of the website.
Steps:
  1. Enter the Downloads section of the site by clicking the "Downloads" tab.
  2. Click the header of the "Filename" column to sort by the file name alphabetically. Choose either "sort up" or "sort down" and remember the choice.
  3. Verify that the list of files is now sorted by file name in the manner that was chosen in step 2.
  4. Now click the "Filename" column header again, choosing the sorting mechanism (i.e., "up" or "down") that was NOT chosen in step 2.
  5. Verify that the list of files is now sorted by file name in the manner that was chosen in step 4.
  6. Click the "Filename" column header again and choose "hide column."
  7. Verify that the Filename column is now hidden.
  8. Repeat steps 2 through 7 with the other columns of the list.

  2. Open the XML file and add a new comment in it.
  3. Re-upload the XML file.
  4. Go to the Issue List and visit the issue's Issue Information page.
  5. Verify that the new comment is shown.

TC-605A: Issue Attribute Change via Web

Purpose: Determine whether issues' information can be changed via the Web interface.
Prereq: Be logged in (see TC-101A) and be a Project Member. Have at least two Issues created.
Steps:
  1. Enter the Administration section of the site.
  2. Choose the Issue Configuration tool.
  3. Define a new tag in the Tags subsection.
  4. Change the column sort order in the column sorting subsection.
  5. Save the changes.
  6. Go to the Issues section of the site.
  7. Verify that the issue list is sorted according to the new specification.
  8. Choose to create a new issue.
  9. Verify that the newly defined tag is an option in the pop-up menu of tags.

TC-700C: Anonymous Subversion Checkout

Purpose: Determine whether the system gives accurate instructions and allows anonymous users to check out files via Subversion.
Prereq: Have a Subversion client.
Steps:
  1. From the project home page, click the "Source" tab.
  2. View the instructions provided on the Subversion repository URL under the section, "Use this command to anonymously check out the latest project source code:"
  3. Attempt to follow the instructions using a Subversion client.
  4. Verify that the Subversion login is successful and the project's repository is downloadable.

TC-700D: Identified Subversion Checkout

Purpose: Determine whether the system allows identified users to log in using a Subversion client.
Prereq: Be a member of a project and have a Subversion client.
Steps:
  1. Log in (see TC-101A).
  2. From the project for which the user is a member, click the "Source" tab.
  3. View the instructions the system provides under the heading, "If you plan to make changes, use this command to check out the code as yourself using HTTPS:".
  4. Follow the link to the user's generated password and record the password externally.
  5. Attempt to log into the Subversion repository using the given URL and username and password with a Subversion client.
  6. Verify that the Subversion client shows that the user is logged in with the given username.

TC-1000: View Profile

Purpose: Determine whether the system correctly displays user profiles.
Prereq: Have a user with a known username that is only a member of one project.
Steps:
  1. Visit the user's project's home page.
  2. Click on the user's home page.
  3. View the user's profile page.
  4. Verify that the user's profile page displays his or her username and only links to the aforementioned project in the "project membership" section.
